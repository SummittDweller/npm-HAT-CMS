# Work Log - 2026-05-04

## Standing Agenda pinned to top of home page

### Problem
The `standing-agenda.md` event page was not reliably appearing at the top of the home page, which sorts content by `publishDate` descending.

### Changes

**`site/content/event/standing-agenda.md`**
- Changed `publishDate` from `now()` to `2099-12-31` so it always sorts above all other content.

**`package.json`**
- Added `-F` (build future) flag to the `build:hugo` script so Hugo includes pages with a future `publishDate` in production builds.
- The `preview:hugo` script already included `-F`; this aligns production with that behavior.
- No other content has a future `publishDate`, so only the standing agenda is affected.
